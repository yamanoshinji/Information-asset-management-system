# ADR 0001: バックエンドフレームワークにNestJSを採用する

## ステータス
Accepted (承認済)

## 背景
フロントエンドをTypeScriptで構築するにあたり、バックエンドの技術選定が必要となった。候補として、Java (Spring Boot)、TypeScript (Express)、TypeScript (NestJS) が挙がった。

## 決定事項
バックエンドフレームワークとして **NestJS** を採用する。

## 理由
1. **型の共有によるCopilotの恩恵最大化**: フロント・バック共にTypeScriptとすることで、APIのインターフェース型を共有でき、AIによるコード生成精度が劇的に向上する。
2. **Spring Bootライクな堅牢性**: Expressの「自由度が高すぎる（スパゲッティコードになりやすい）」というデメリットに対し、NestJSはモジュール・コントローラー・サービスの層が明確に分かれており、チーム開発や将来の拡張に耐えうるエンタープライズな設計が可能であるため。
