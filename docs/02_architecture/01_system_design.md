# システムアーキテクチャ設計

## 1. 全体方針
- オンプレミスへの移行も容易に行えるよう、**Dockerによるコンテナベースのアーキテクチャ**を採用する。
- 開発効率とGitHub Copilotのサジェスト精度を最大化するため、フロントエンド・バックエンド共に**TypeScript**で統一する。

## 2. 技術スタック
| 領域 | 採用技術 | 選定理由・備考 |
| :--- | :--- | :--- |
| **フロントエンド** | TypeScript, React, MUI (or shadcn/ui) | 型安全なUI構築。管理画面の要件に適したコンポーネント指向。 |
| **バックエンド** | TypeScript, NestJS | 堅牢なアーキテクチャ(DI)、REST APIの構築、Swaggerによる型自動生成が強力。 |
| **ORM** | Prisma | DBスキーマからTypeScriptの型を自動生成し、Copilotと極めて相性が良い。 |
| **データベース** | PostgreSQL | `JSONB` 型を活用した柔軟なデータ構造（カテゴリごとの個別項目管理）を実現するため。 |
| **インフラ** | Docker (AWS ECS等想定) | ベンダーロックインを回避し、将来的なオンプレミス環境での稼働を保証する。 |
| **API設計** | REST API | スタンダードなCRUD操作に最適であり、開発スピードが最速のため。 |
